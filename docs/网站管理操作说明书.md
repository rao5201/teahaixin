## 1. 网站概述

茶海心遇的网站包含两个独立部分：

| 部分 | 目录 | 用途 | 技术 |
|------|------|------|------|
| 宣传官网 | `website/` | 产品展示、功能介绍、下载引导 | 静态 HTML 单页面 |
| PWA 应用 | `webapp/` | 用户实际使用的网页版应用 | HTML + JS + Service Worker |

两者可以独立部署在不同域名下，也可以部署在同一域名的不同路径下。

## 2. GitHub Pages 配置与管理

### 2.1 开启/关闭 Pages

**开启 GitHub Pages：**

1. 进入 GitHub 仓库页面
2. 点击 **Settings** 标签
3. 左侧菜单选择 **Pages**
4. 在 "Source" 下拉框中选择部署来源：
   - **Deploy from a branch** - 从指定分支部署
   - 选择分支（通常为 `main`）和目录（`/ (root)` 或 `/docs`）
5. 点击 **Save**
6. 等待几分钟后即可通过 `https://<用户名>.github.io/<仓库名>/` 访问

**关闭 GitHub Pages：**

1. 进入 Settings > Pages
2. 在 "Source" 下拉框中选择 **None**
3. 点击 **Save**

### 2.2 自定义域名配置

#### 步骤一：DNS 配置

在你的域名服务商处添加 DNS 记录：

| 记录类型 | 主机名 | 值 |
|----------|--------|-----|
| CNAME | www | `<用户名>.github.io` |
| A | @ | `185.199.108.153` |
| A | @ | `185.199.109.153` |
| A | @ | `185.199.110.153` |
| A | @ | `185.199.111.153` |

#### 步骤二：GitHub 配置

1. 进入 Settings > Pages
2. 在 "Custom domain" 输入框中填入你的域名（如 `www.teahaixin.com`）
3. 点击 **Save**
4. 等待 DNS 生效（可能需要几分钟到几小时）

#### 步骤三：创建 CNAME 文件

在部署的根目录下创建 `CNAME` 文件，内容为你的自定义域名：

```
www.teahaixin.com
```

### 2.3 HTTPS 设置

1. 进入 Settings > Pages
2. 勾选 **Enforce HTTPS**
3. GitHub 会自动为你的域名申请和续期 SSL 证书（Let's Encrypt）

> **注意：** 自定义域名的 HTTPS 证书颁发可能需要最多 24 小时。在此期间可能出现证书警告。

### 2.4 部署流程

GitHub Pages 支持自动部署，流程如下：

```
本地修改文件 → git add → git commit → git push → GitHub Pages 自动构建部署
```

**具体步骤：**

```bash
# 1. 修改网站文件
# 2. 提交更改
git add website/ webapp/
git commit -m "更新网站内容"

# 3. 推送到远程仓库
git push origin main

# 4. GitHub Actions 自动部署（约 1-3 分钟）
```

**验证部署：**

- 访问 GitHub 仓库的 **Actions** 标签查看部署状态
- 部署完成后访问网站 URL 确认更新生效

## 3. 宣传官网管理

### 3.1 文件结构

宣传官网采用单文件结构，所有内容集中在一个 HTML 文件中：

```
website/
└── index.html          # 包含 HTML 结构、CSS 样式、全部内容
```

页面主要区域：

| 区域 | 说明 |
|------|------|
| Header/导航栏 | 品牌 Logo、导航链接 |
| Hero 区域 | 品牌 Slogan、主要行动按钮 |
| 功能介绍 | 核心功能展示卡片 |
| 产品截图/演示 | 应用界面展示 |
| 用户评价 | 用户好评展示 |
| FAQ 问答 | 常见问题解答 |
| 下载/使用引导 | 下载链接和使用入口 |
| Footer 页脚 | 版权信息、联系方式 |

### 3.2 修改品牌信息

打开 `website/index.html`，搜索并修改以下内容：

#### 修改品牌名称

搜索"茶海心遇"，替换为新的品牌名称。需要修改的位置包括：
- `<title>` 标签
- `<meta name="description">` 标签
- 页面内所有品牌名称文字

#### 修改 Slogan

搜索"以茶会心，遇见情绪"，替换为新的 Slogan。

#### 修改页面描述

```html
<!-- 修改 meta description -->
<meta name="description" content="一款AI驱动的情绪可视化社交APP，将你的心情转化为唯美的视觉卡片与音乐">
```

### 3.3 修改功能介绍内容

功能介绍通常以卡片形式展示。在 `index.html` 中找到功能介绍区域，修改对应的标题、描述文字和图标。

**示例结构：**

```html
<div class="feature-card">
    <div class="feature-icon">图标</div>
    <h3>功能标题</h3>
    <p>功能描述文字</p>
</div>
```

修改时保持 HTML 结构不变，仅修改文字内容即可。

### 3.4 更新下载链接

在 `index.html` 中找到下载按钮区域，修改链接地址：

```html
<!-- PWA 网页版链接 -->
<a href="https://你的域名/webapp/">立即体验 PWA 版</a>

<!-- APK 下载链接 -->
<a href="https://下载地址/teahaixin.apk">下载 Android 版</a>
```

### 3.5 更新用户评价

在 `index.html` 中找到用户评价区域，修改或添加评价内容：

```html
<div class="testimonial">
    <p class="quote">"评价内容"</p>
    <div class="author">
        <span class="name">用户名</span>
        <span class="role">用户身份描述</span>
    </div>
</div>
```

### 3.6 修改 FAQ 内容

在 `index.html` 中找到 FAQ 区域，修改问答对：

```html
<div class="faq-item">
    <h4 class="faq-question">问题标题</h4>
    <p class="faq-answer">回答内容</p>
</div>
```

**添加新的 FAQ：** 复制上述结构，修改内容后粘贴到 FAQ 列表中即可。

### 3.7 样式调整

所有样式定义在 `index.html` 的 `<style>` 标签内。

#### 修改主题色

搜索以下颜色值进行替换：

| 用途 | 当前值 | 说明 |
|------|--------|------|
| 主色调 | `#6c5ce7` | 紫色，用于按钮、链接、高亮 |
| 主色浅色 | `#a29bfe` | 浅紫色，用于渐变和次要元素 |
| 背景色 | `#0a0a14` | 深色背景 |
| 文字色 | `#e0dde5` | 主要文字颜色 |
| 卡片背景 | `#1e1e2f` | 卡片和模块背景色 |

#### 修改字体

```css
body {
  font-family: "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", "Noto Serif SC", serif;
}
```

可根据需要调整字体优先级或添加新字体。

#### 修改间距

全局间距通过 CSS 类控制：

```css
.container { max-width: 1200px; margin: 0 auto; padding: 0 24px; }
.section-title { margin-bottom: 1rem; }
```

## 4. PWA 应用管理

### 4.1 webapp 目录结构说明

```
webapp/
├── index.html          # PWA 主页面（HTML 结构和 CSS 样式）
├── app.js              # 应用逻辑（情绪分析、结果生成、导航等）
├── sw.js               # Service Worker（离线缓存和资源管理）
├── manifest.json       # PWA 配置清单（应用名称、图标、主题色）
├── icons/              # 应用图标（不同尺寸）
│   ├── icon-192.png
│   └── icon-512.png
└── style.css           # 样式文件（如有独立样式文件）
```

### 4.2 修改应用配置 manifest.json

`manifest.json` 控制 PWA 安装后的应用表现：

```json
{
  "name": "茶海心遇",
  "short_name": "茶海心遇",
  "description": "AI情绪可视化社交APP",
  "start_url": "./",
  "display": "standalone",
  "background_color": "#0f0f1a",
  "theme_color": "#6c5ce7",
  "icons": [
    {
      "src": "icons/icon-192.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "icons/icon-512.png",
      "sizes": "512x512",
      "type": "image/png"
    }
  ]
}
```

**可修改字段说明：**

| 字段 | 说明 |
|------|------|
| `name` | 安装后显示的完整应用名称 |
| `short_name` | 主屏幕图标下方显示的简短名称 |
| `description` | 应用描述 |
| `start_url` | 应用启动时加载的页面 |
| `display` | 显示模式，`standalone` 表示独立应用窗口 |
| `background_color` | 启动画面背景色 |
| `theme_color` | 系统状态栏主题色 |
| `icons` | 不同尺寸的应用图标 |

> **注意：** 修改 `manifest.json` 后，用户需要重新安装 PWA 才能看到更新效果。

### 4.3 更新 Service Worker 缓存策略

Service Worker 文件 `sw.js` 控制资源的离线缓存策略。

**更新缓存版本：**

每次更新网站内容后，需要更新 Service Worker 中的缓存版本号，以确保用户获取最新资源：

```javascript
// sw.js 中修改版本号
const CACHE_NAME = 'teahaixin-v2';  // 递增版本号
```

**自定义缓存资源列表：**

```javascript
const ASSETS_TO_CACHE = [
  './',
  './index.html',
  './app.js',
  './style.css',
  './manifest.json',
  './icons/icon-192.png',
  './icons/icon-512.png'
];
```

**缓存策略选择：**

| 策略 | 适用场景 | 说明 |
|------|----------|------|
| Cache First | 静态资源（CSS、JS、图片） | 优先从缓存读取，缓存未命中时走网络 |
| Network First | API 请求、动态内容 | 优先走网络，网络失败时回退缓存 |
| Stale While Revalidate | 不频繁更新的资源 | 立即返回缓存，同时后台更新缓存 |

### 4.4 修改情绪关键词库

情绪关键词库定义在 `webapp/app.js` 的 `KEYWORDS` 对象中，用于本地情绪分析：

```javascript
const KEYWORDS = {
  happy: ['开心','快乐','高兴','幸福','喜','棒','好','美','笑','阳光',...],
  sad: ['难过','伤心','悲','哭','泪','思念','想念','离别','孤独',...],
  angry: ['生气','愤怒','烦','恨','气','火','怒','讨厌',...],
  peaceful: ['平静','安宁','静','淡','闲','慢','舒适','放松',...],
  anxious: ['焦虑','紧张','担心','害怕','恐惧','不安','压力',...],
  romantic: ['爱','恋','喜欢','心动','浪漫','甜蜜','温柔',...]
};
```

**添加新关键词：**

1. 打开 `webapp/app.js`
2. 找到 `KEYWORDS` 对象
3. 在对应情绪类型的数组中添加新关键词
4. 同时支持中文和英文关键词

**添加新的情绪类型（如需扩展）：**

1. 在 `KEYWORDS` 中添加新类型及其关键词
2. 在 `EMOTIONS` 对象中添加对应的标签、表情、颜色和图片提示词
3. 在 `POEMS` 对象中添加对应的诗词
4. 在 `AUDIO_URLS` 中添加对应的音频 URL
5. 在 `AUDIO_LABELS` 中添加音频标签

### 4.5 添加新的诗词模板

诗词模板定义在 `webapp/app.js` 的 `POEMS` 对象中：

```javascript
const POEMS = {
  happy: [
    '春风得意马蹄疾，\n一日看尽长安花。\n心随白云自在飞，\n茶香满盏笑颜开。',
    // ... 更多诗词
  ],
  // ... 其他情绪类型
};
```

**添加新诗词步骤：**

1. 打开 `webapp/app.js`
2. 找到 `POEMS` 对象
3. 在对应情绪类型的数组中添加新的诗词字符串
4. 使用 `\n` 表示换行
5. 建议每种情绪至少保留 3-4 首诗词以保证随机性

**诗词格式建议：**

- 每首诗 4 行为佳（四言或七言）
- 内容与情绪类型贴合
- 融入茶文化元素更佳
- 支持经典诗词和原创内容

### 4.6 修改音频资源

音频 URL 定义在 `webapp/app.js` 的 `AUDIO_URLS` 对象中：

```javascript
const AUDIO_URLS = {
  happy: 'https://cdn.pixabay.com/audio/2022/10/18/audio_a12a067cfc.mp3',
  sad: 'https://cdn.pixabay.com/audio/2022/01/20/audio_d1718ab41b.mp3',
  angry: 'https://cdn.pixabay.com/audio/2021/08/08/audio_dc39537e1a.mp3',
  peaceful: 'https://cdn.pixabay.com/audio/2022/03/15/audio_115fe72651.mp3',
  anxious: 'https://cdn.pixabay.com/audio/2022/05/16/audio_4e2bfab09c.mp3',
  romantic: 'https://cdn.pixabay.com/audio/2022/08/31/audio_419263edd1.mp3'
};
```

**替换音频步骤：**

1. 准备新的音频文件（推荐 MP3 格式，时长 1-3 分钟，文件大小控制在 5MB 以内）
2. 将音频上传到 CDN 或静态资源服务器
3. 获取可公开访问的 URL
4. 替换 `AUDIO_URLS` 中对应情绪类型的 URL

**音频选择建议：**

| 情绪 | 音乐风格建议 |
|------|-------------|
| 喜悦 | 轻快、明亮、欢快的旋律 |
| 忧伤 | 缓慢、低沉、略带忧伤的钢琴或弦乐 |
| 愤怒 | 激昂、有力、节奏强烈的乐章 |
| 平和 | 自然之声、古琴、禅意音乐 |
| 焦虑 | 舒缓、治愈、冥想类音乐 |
| 浪漫 | 温柔、甜蜜、抒情的旋律 |

> **注意：** 确保使用的音频资源符合版权要求。Pixabay 提供免费可商用的音频资源。

## 5. SEO 优化建议

### 基础 SEO 配置

确保 `website/index.html` 中包含以下 meta 标签：

```html
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>茶海心遇 TeaHaiXin - 以茶会心，遇见情绪</title>
  <meta name="description" content="一款AI驱动的情绪可视化社交APP，将你的心情转化为唯美的视觉卡片与音乐">
  <meta name="keywords" content="茶海心遇,TeaHaiXin,情绪,AI,心情,诗词,情绪可视化,社交,茶文化">

  <!-- Open Graph（社交分享优化） -->
  <meta property="og:title" content="茶海心遇 - 以茶会心，遇见情绪">
  <meta property="og:description" content="AI驱动的情绪可视化社交APP">
  <meta property="og:type" content="website">
  <meta property="og:url" content="https://www.teahaixin.com">
  <meta property="og:image" content="https://www.teahaixin.com/og-image.jpg">

  <!-- Twitter Card -->
  <meta name="twitter:card" content="summary_large_image">
  <meta name="twitter:title" content="茶海心遇 - 以茶会心，遇见情绪">
  <meta name="twitter:description" content="AI驱动的情绪可视化社交APP">
</head>
```

### 进阶 SEO 建议

| 优化项 | 说明 |
|--------|------|
| 语义化 HTML | 使用 `<header>`, `<main>`, `<section>`, `<article>`, `<footer>` 等语义标签 |
| 标题层级 | 保持 h1 > h2 > h3 的层级结构，每页仅一个 h1 |
| 图片优化 | 为所有 `<img>` 添加 `alt` 属性，使用 WebP 格式减小体积 |
| 页面加载速度 | 压缩 CSS/JS，启用 GZIP，使用 CDN 加速 |
| 移动端适配 | 确保响应式设计，通过 Google Mobile-Friendly Test |
| 结构化数据 | 添加 JSON-LD 结构化数据标记 |
| robots.txt | 在根目录创建 `robots.txt` 允许搜索引擎爬取 |
| sitemap.xml | 创建站点地图帮助搜索引擎索引 |

### robots.txt 示例

```
User-agent: *
Allow: /
Sitemap: https://www.teahaixin.com/sitemap.xml
```

## 6. 访问统计集成

### Google Analytics 集成

在 `website/index.html` 的 `<head>` 标签内添加：

```html
<!-- Google Analytics 4 -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX');
</script>
```

将 `G-XXXXXXXXXX` 替换为你的 Google Analytics 测量 ID。

### 百度统计集成

```html
<script>
var _hmt = _hmt || [];
(function() {
  var hm = document.createElement("script");
  hm.src = "https://hm.baidu.com/hm.js?你的百度统计ID";
  var s = document.getElementsByTagName("script")[0];
  s.parentNode.insertBefore(hm, s);
})();
</script>
```

### 关注的核心指标

| 指标 | 说明 |
|------|------|
| 页面浏览量 (PV) | 总访问次数 |
| 独立访客数 (UV) | 去重的访问用户数 |
| 跳出率 | 只看一个页面就离开的比例 |
| 平均停留时间 | 用户在网站上的平均停留时长 |
| PWA 安装率 | 点击安装/访问总数 |
| 下载转化率 | 点击下载按钮/访问总数 |
| 设备分布 | 移动端 vs 桌面端访问占比 |
| 地域分布 | 用户来源地区 |

## 7. 版本更新流程

### 更新宣传官网

```bash
# 1. 修改 website/index.html 中的内容
# 2. 本地预览（可用任意 HTTP 服务器）
cd website
npx serve .
# 或
python3 -m http.server 8000
```

### 版本号管理建议

采用语义化版本号 `主版本.次版本.修订号`：

| 变更类型 | 版本号变化 | 示例 |
|----------|-----------|------|
| 重大功能更新或不兼容变更 | 主版本号 +1 | 1.0.0 -> 2.0.0 |
| 新增功能（向下兼容） | 次版本号 +1 | 1.0.0 -> 1.1.0 |
| 修复 Bug 或小幅调整 | 修订号 +1 | 1.0.0 -> 1.0.1 |
