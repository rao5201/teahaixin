## 1. 数据库概述

茶海心遇使用 **SQLite** 作为数据库引擎，通过 **better-sqlite3** Node.js 驱动进行操作。

| 属性 | 值 |
|------|-----|
| 数据库引擎 | SQLite 3 |
| Node.js 驱动 | better-sqlite3 |
| 数据库文件路径 | `backend/data/teahaixin.db` |
| 日志模式 | WAL（Write-Ahead Logging） |
| 外键约束 | 已启用（`PRAGMA foreign_keys = ON`） |

**数据库初始化流程：**

服务器启动时，`backend/config/database.js` 中的 `initDatabase()` 函数会自动执行以下操作：
1. 创建数据库文件（如不存在）
2. 启用 WAL 日志模式
3. 启用外键约束
4. 创建所有数据表（`CREATE TABLE IF NOT EXISTS`）
5. 创建所有索引（`CREATE INDEX IF NOT EXISTS`）

**访问数据库：**

```bash
# 使用 sqlite3 命令行工具
sqlite3 backend/data/teahaixin.db

# 进入后可执行 SQL 查询
sqlite> .tables
sqlite> .schema users
sqlite> SELECT COUNT(*) FROM users;
sqlite> .quit
```

## 2. 数据表结构

### 2.1 users 表 - 用户信息

存储所有注册用户的基本信息。

**建表语句：**

```sql
CREATE TABLE IF NOT EXISTS users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT UNIQUE NOT NULL,
    password TEXT NOT NULL,
    avatar TEXT DEFAULT '',
    created_at DATETIME DEFAULT (datetime('now', 'localtime'))
);
```

**字段说明：**

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 自增 | 用户唯一标识 |
| username | TEXT | UNIQUE, NOT NULL | 无 | 用户名，2-20个字符，全局唯一 |
| password | TEXT | NOT NULL | 无 | 密码，使用 bcrypt（10轮盐值）加密存储 |
| avatar | TEXT | 无 | '' | 用户头像，注册时随机分配一个表情符号 |
| created_at | DATETIME | 无 | 当前本地时间 | 注册时间 |

**头像可选值：**

```
🍵 🌸 🌙 🌊 🎋 🏔️ 🌿 🎑 ☁️ 🌺
🦋 🍃 🌅 🎐 🪷 🌻 🐚 🌈 🕊️ 🌾
```

### 2.2 emotions 表 - 情绪记录

存储用户的所有情绪记录及 AI 生成结果。

**建表语句：**

```sql
CREATE TABLE IF NOT EXISTS emotions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    text TEXT NOT NULL,
    emotion TEXT NOT NULL,
    image_url TEXT DEFAULT '',
    audio_url TEXT DEFAULT '',
    poem_text TEXT DEFAULT '',
    location TEXT DEFAULT '',
    latitude REAL DEFAULT 0,
    longitude REAL DEFAULT 0,
    is_public INTEGER DEFAULT 1,
    created_at DATETIME DEFAULT (datetime('now', 'localtime')),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE SET NULL
);
```

**字段说明：**

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 自增 | 记录唯一标识 |
| user_id | INTEGER | FOREIGN KEY -> users(id) | 无 | 所属用户 ID，用户删除时设为 NULL |
| text | TEXT | NOT NULL | 无 | 用户输入的心情文字 |
| emotion | TEXT | NOT NULL | 无 | 情绪类型：happy/sad/angry/peaceful/anxious/romantic |
| image_url | TEXT | 无 | '' | AI 生成的意境图片 URL |
| audio_url | TEXT | 无 | '' | 情绪背景音乐 URL |
| poem_text | TEXT | 无 | '' | 匹配的诗意文本 |
| location | TEXT | 无 | '' | 位置描述文字（如"北京朝阳区"） |
| latitude | REAL | 无 | 0 | 纬度坐标，0 表示未设置 |
| longitude | REAL | 无 | 0 | 经度坐标，0 表示未设置 |
| is_public | INTEGER | 无 | 1 | 是否公开：1=公开，0=私密 |
| created_at | DATETIME | 无 | 当前本地时间 | 记录创建时间 |

**emotion 字段取值：**

| 值 | 中文标签 | 说明 |
|----|----------|------|
| happy | 喜悦 | 开心、快乐、幸福等积极情绪 |
| sad | 忧伤 | 难过、伤心、孤独等消极情绪 |
| angry | 愤怒 | 生气、烦躁、不满等激烈情绪 |
| peaceful | 平和 | 平静、安宁、放松等中性情绪 |
| anxious | 焦虑 | 紧张、担心、不安等压力情绪 |
| romantic | 浪漫 | 爱意、心动、甜蜜等浪漫情绪 |

### 2.3 resonances 表 - 共振记录

存储用户之间的情绪共振连接。

**建表语句：**

```sql
CREATE TABLE IF NOT EXISTS resonances (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    from_user_id INTEGER NOT NULL,
    to_user_id INTEGER NOT NULL,
    emotion_id INTEGER NOT NULL,
    created_at DATETIME DEFAULT (datetime('now', 'localtime')),
    FOREIGN KEY (from_user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (to_user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (emotion_id) REFERENCES emotions(id) ON DELETE CASCADE
);
```

**字段说明：**

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 自增 | 共振记录唯一标识 |
| from_user_id | INTEGER | NOT NULL, FK -> users(id) | 无 | 发起共振的用户 ID，用户删除时级联删除 |
| to_user_id | INTEGER | NOT NULL, FK -> users(id) | 无 | 共振目标用户 ID，用户删除时级联删除 |
| emotion_id | INTEGER | NOT NULL, FK -> emotions(id) | 无 | 关联的情绪记录 ID，记录删除时级联删除 |
| created_at | DATETIME | 无 | 当前本地时间 | 共振建立时间 |

**外键级联行为：**

| 外键 | 删除策略 | 说明 |
|------|----------|------|
| emotions.user_id -> users.id | SET NULL | 用户删除后，情绪记录保留但 user_id 置空 |
| resonances.from_user_id -> users.id | CASCADE | 用户删除后，相关共振记录一并删除 |
| resonances.to_user_id -> users.id | CASCADE | 用户删除后，相关共振记录一并删除 |
| resonances.emotion_id -> emotions.id | CASCADE | 情绪记录删除后，相关共振记录一并删除 |

## 3. 索引说明

系统在初始化时自动创建以下索引以提升查询性能：

```sql
CREATE INDEX IF NOT EXISTS idx_emotions_user_id ON emotions(user_id);
CREATE INDEX IF NOT EXISTS idx_emotions_is_public ON emotions(is_public);
CREATE INDEX IF NOT EXISTS idx_emotions_created_at ON emotions(created_at);
CREATE INDEX IF NOT EXISTS idx_emotions_location ON emotions(latitude, longitude);
CREATE INDEX IF NOT EXISTS idx_resonances_to_user ON resonances(to_user_id);
CREATE INDEX IF NOT EXISTS idx_resonances_from_user ON resonances(from_user_id);
```

**索引用途说明：**

| 索引名 | 字段 | 用途 |
|--------|------|------|
| idx_emotions_user_id | emotions(user_id) | 加速按用户查询情绪记录（历史记录、周报） |
| idx_emotions_is_public | emotions(is_public) | 加速公开情绪列表查询 |
| idx_emotions_created_at | emotions(created_at) | 加速按时间排序和范围查询 |
| idx_emotions_location | emotions(latitude, longitude) | 加速地理位置范围查询（附近情绪、热力图） |
| idx_resonances_to_user | resonances(to_user_id) | 加速查询收到的共振记录 |
| idx_resonances_from_user | resonances(from_user_id) | 加速查询发出的共振记录 |

## 4. 常用查询操作

以下所有 SQL 示例均可在 `sqlite3` 命令行工具中直接执行。

### 4.1 用户管理查询

#### 查看所有用户

```sql
SELECT id, username, avatar, created_at
FROM users
ORDER BY created_at DESC;
```

#### 查看用户总数

```sql
SELECT COUNT(*) AS total_users FROM users;
```

#### 查找指定用户

```sql
-- 按用户名查找
SELECT * FROM users WHERE username = '茶友小明';

-- 按 ID 查找
SELECT id, username, avatar, created_at FROM users WHERE id = 1;
```

#### 查看最近注册的用户

```sql
SELECT id, username, avatar, created_at
FROM users
ORDER BY created_at DESC
LIMIT 10;
```

#### 查看用户的活跃统计

```sql
SELECT
    u.id,
    u.username,
    COUNT(e.id) AS emotion_count,
    COUNT(DISTINCT e.emotion) AS emotion_types,
    COUNT(DISTINCT date(e.created_at)) AS active_days
FROM users u
LEFT JOIN emotions e ON u.id = e.user_id
GROUP BY u.id
ORDER BY emotion_count DESC;
```

### 4.2 情绪记录查询

#### 查看指定用户的情绪记录

```sql
SELECT e.id, e.text, e.emotion, e.poem_text, e.location, e.created_at
FROM emotions e
WHERE e.user_id = 1
ORDER BY e.created_at DESC
LIMIT 20;
```

#### 按情绪类型筛选

```sql
SELECT e.*, u.username
FROM emotions e
LEFT JOIN users u ON e.user_id = u.id
WHERE e.emotion = 'happy' AND e.is_public = 1
ORDER BY e.created_at DESC
LIMIT 20;
```

#### 按日期范围查询

```sql
-- 查看最近7天的情绪记录
SELECT *
FROM emotions
WHERE created_at >= datetime('now', '-7 days', 'localtime')
ORDER BY created_at DESC;

-- 查看指定日期范围
SELECT *
FROM emotions
WHERE created_at BETWEEN '2026-03-01 00:00:00' AND '2026-03-31 23:59:59'
ORDER BY created_at DESC;
```

#### 按情绪类型统计

```sql
SELECT
    emotion,
    COUNT(*) AS count,
    ROUND(COUNT(*) * 100.0 / (SELECT COUNT(*) FROM emotions), 2) AS percentage
FROM emotions
GROUP BY emotion
ORDER BY count DESC;
```

#### 查看带有位置信息的记录

```sql
SELECT id, text, emotion, location, latitude, longitude, created_at
FROM emotions
WHERE latitude != 0 AND longitude != 0
ORDER BY created_at DESC
LIMIT 50;
```

#### 按用户统计情绪记录数

```sql
SELECT
    u.username,
    COUNT(e.id) AS total_records,
    MAX(e.created_at) AS last_record_time
FROM users u
LEFT JOIN emotions e ON u.id = e.user_id
GROUP BY u.id
ORDER BY total_records DESC
LIMIT 20;
```

### 4.3 共振记录查询

#### 查看所有共振记录

```sql
SELECT
    r.id,
    fu.username AS from_user,
    tu.username AS to_user,
    e.emotion,
    e.text AS emotion_text,
    r.created_at
FROM resonances r
JOIN users fu ON r.from_user_id = fu.id
JOIN users tu ON r.to_user_id = tu.id
JOIN emotions e ON r.emotion_id = e.id
ORDER BY r.created_at DESC
LIMIT 50;
```

#### 查看指定用户的共振

```sql
-- 查看用户 ID=1 的所有共振
SELECT
    r.*,
    fu.username AS from_user,
    tu.username AS to_user,
    e.emotion
FROM resonances r
JOIN users fu ON r.from_user_id = fu.id
JOIN users tu ON r.to_user_id = tu.id
JOIN emotions e ON r.emotion_id = e.id
WHERE r.from_user_id = 1 OR r.to_user_id = 1
ORDER BY r.created_at DESC;
```

#### 共振统计

```sql
SELECT
    e.emotion,
    COUNT(*) AS resonance_count
FROM resonances r
JOIN emotions e ON r.emotion_id = e.id
GROUP BY e.emotion
ORDER BY resonance_count DESC;
```

### 4.4 统计分析查询

#### 每日情绪记录统计

```sql
SELECT
    date(created_at) AS date,
    COUNT(*) AS daily_count
FROM emotions
WHERE created_at >= datetime('now', '-30 days', 'localtime')
GROUP BY date(created_at)
ORDER BY date;
```

#### 每日各情绪类型分布

```sql
SELECT
    date(created_at) AS date,
    emotion,
    COUNT(*) AS count
FROM emotions
WHERE created_at >= datetime('now', '-7 days', 'localtime')
GROUP BY date(created_at), emotion
ORDER BY date, count DESC;
```

#### 用户活跃度排行

```sql
SELECT
    u.username,
    COUNT(DISTINCT date(e.created_at)) AS active_days,
    COUNT(e.id) AS total_records,
    COUNT(r_sent.id) AS resonances_sent,
    COUNT(r_recv.id) AS resonances_received
FROM users u
LEFT JOIN emotions e ON u.id = e.user_id
LEFT JOIN resonances r_sent ON u.id = r_sent.from_user_id
LEFT JOIN resonances r_recv ON u.id = r_recv.to_user_id
GROUP BY u.id
ORDER BY total_records DESC
LIMIT 20;
```

### 4.5 热力图数据查询

#### 查看情绪热力图数据

```sql
SELECT
    ROUND(latitude, 2) AS lat,
    ROUND(longitude, 2) AS lng,
    emotion,
    COUNT(*) AS intensity
FROM emotions
WHERE is_public = 1
    AND latitude != 0
    AND longitude != 0
    AND created_at >= datetime('now', '-7 days', 'localtime')
GROUP BY ROUND(latitude, 2), ROUND(longitude, 2), emotion
ORDER BY intensity DESC
LIMIT 500;
```

#### 查看情绪地理分布汇总

```sql
SELECT
    emotion,
    COUNT(*) AS count,
    AVG(latitude) AS avg_lat,
    AVG(longitude) AS avg_lng
FROM emotions
WHERE is_public = 1
    AND latitude != 0
    AND longitude != 0
GROUP BY emotion
ORDER BY count DESC;
```

## 5. 数据备份与恢复

### 备份数据库

SQLite 数据库是单个文件，备份操作非常简单：

```bash
# 方式一：直接复制文件（确保无写入操作时执行）
cp backend/data/teahaixin.db backend/data/teahaixin_backup_$(date +%Y%m%d_%H%M%S).db

# 方式二：使用 sqlite3 的 .backup 命令（推荐，支持热备份）
sqlite3 backend/data/teahaixin.db ".backup backend/data/teahaixin_backup.db"
```

### 自动定期备份脚本

创建 `backup.sh` 脚本：

```bash
#!/bin/bash
BACKUP_DIR="/path/to/teahaixin/backend/data/backups"
DB_PATH="/path/to/teahaixin/backend/data/teahaixin.db"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)

mkdir -p "$BACKUP_DIR"
sqlite3 "$DB_PATH" ".backup $BACKUP_DIR/teahaixin_$TIMESTAMP.db"

# 保留最近 30 天的备份，删除更早的
find "$BACKUP_DIR" -name "teahaixin_*.db" -mtime +30 -delete

echo "[$(date)] 备份完成: $BACKUP_DIR/teahaixin_$TIMESTAMP.db"
```

配置 crontab 每天自动执行：

```bash
# 每天凌晨 3 点执行备份
0 3 * * * /path/to/backup.sh >> /var/log/teahaixin_backup.log 2>&1
```

### 恢复数据库

```bash
# 1. 停止后端服务
pm2 stop teahaixin-api

# 2. 备份当前数据库（以防万一）
cp backend/data/teahaixin.db backend/data/teahaixin_before_restore.db

# 3. 用备份文件替换当前数据库
cp backend/data/backups/teahaixin_20260325_030000.db backend/data/teahaixin.db

# 4. 重新启动服务
pm2 start teahaixin-api
```

## 6. 数据清理

### 删除过期记录

```sql
-- 删除超过 90 天的情绪记录（注意：关联的共振记录会级联删除）
DELETE FROM emotions
WHERE created_at < datetime('now', '-90 days', 'localtime');

-- 删除超过 180 天的旧记录
DELETE FROM emotions
WHERE created_at < datetime('now', '-180 days', 'localtime');
```

### 清空测试数据

```sql
-- 清空所有数据（谨慎操作！）
-- 需要按照外键依赖顺序删除
DELETE FROM resonances;
DELETE FROM emotions;
DELETE FROM users;

-- 重置自增 ID
DELETE FROM sqlite_sequence WHERE name IN ('users', 'emotions', 'resonances');
```

### 删除指定用户及其数据

```sql
-- 查看要删除的用户
SELECT * FROM users WHERE username = '测试用户';

-- 删除用户（共振记录会级联删除，情绪记录的 user_id 会设为 NULL）
DELETE FROM users WHERE username = '测试用户';

-- 如果也要删除该用户的情绪记录
DELETE FROM emotions WHERE user_id IS NULL;
```

### 清理无效数据

```sql
-- 删除没有用户关联的情绪记录（孤立记录）
DELETE FROM emotions WHERE user_id IS NULL;

-- 删除关联情绪记录已不存在的共振
DELETE FROM resonances
WHERE emotion_id NOT IN (SELECT id FROM emotions);
```

### 清理后执行 VACUUM

清理大量数据后，执行 VACUUM 回收磁盘空间：

```bash
sqlite3 backend/data/teahaixin.db "VACUUM;"
```

## 7. 数据迁移

### SQLite 转 MySQL

#### 步骤一：导出数据

```bash
# 导出表结构和数据为 SQL
sqlite3 backend/data/teahaixin.db .dump > teahaixin_dump.sql
```

#### 步骤二：修改 SQL 语法

SQLite 与 MySQL 存在语法差异，需要做以下调整：

| SQLite 语法 | MySQL 语法 |
|-------------|------------|
| `INTEGER PRIMARY KEY AUTOINCREMENT` | `INT PRIMARY KEY AUTO_INCREMENT` |
| `TEXT` | `VARCHAR(n)` 或 `TEXT` |
| `REAL` | `DOUBLE` 或 `DECIMAL` |
| `datetime('now', 'localtime')` | `NOW()` |
| `datetime('now', '-7 days', 'localtime')` | `DATE_SUB(NOW(), INTERVAL 7 DAY)` |

#### 步骤三：MySQL 建表语句参考

```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(20) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    avatar VARCHAR(50) DEFAULT '',
    created_at DATETIME DEFAULT NOW()
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE TABLE emotions (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT,
    text TEXT NOT NULL,
    emotion VARCHAR(20) NOT NULL,
    image_url TEXT DEFAULT '',
    audio_url TEXT DEFAULT '',
    poem_text TEXT DEFAULT '',
    location VARCHAR(255) DEFAULT '',
    latitude DOUBLE DEFAULT 0,
    longitude DOUBLE DEFAULT 0,
    is_public TINYINT DEFAULT 1,
    created_at DATETIME DEFAULT NOW(),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE TABLE resonances (
    id INT PRIMARY KEY AUTO_INCREMENT,
    from_user_id INT NOT NULL,
    to_user_id INT NOT NULL,
    emotion_id INT NOT NULL,
    created_at DATETIME DEFAULT NOW(),
    FOREIGN KEY (from_user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (to_user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (emotion_id) REFERENCES emotions(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 步骤四：代码层修改

将 `better-sqlite3` 替换为 `mysql2` 或 `sequelize`，修改 `config/database.js` 和所有路由文件中的数据库操作代码。

### SQLite 转 PostgreSQL

流程与转 MySQL 类似，主要语法差异：

| SQLite 语法 | PostgreSQL 语法 |
|-------------|----------------|
| `INTEGER PRIMARY KEY AUTOINCREMENT` | `SERIAL PRIMARY KEY` |
| `TEXT` | `TEXT` 或 `VARCHAR(n)` |
| `REAL` | `DOUBLE PRECISION` |
| `datetime('now', 'localtime')` | `NOW()` |

### 迁移建议

- 数据量小于 10 万条记录时，SQLite 性能足够，无需迁移
- 当并发用户超过 50 或数据量快速增长时，建议迁移到 MySQL/PostgreSQL
- 迁移前务必完整备份原始数据库
- 迁移后进行全面的接口测试

## 8. 安全注意事项

### 密码加密

系统使用 **bcryptjs** 进行密码加密：

```javascript
// 加密（10轮盐值）
const hashedPassword = bcrypt.hashSync(password, 10);

// 验证
const isValid = bcrypt.compareSync(plainPassword, hashedPassword);
```

- 密码在存储前经过 10 轮 bcrypt 盐值加密
- 即使数据库泄露，攻击者也无法还原原始密码
- bcrypt 的计算成本随轮数指数增长，有效防御暴力破解

### SQL 注入防护

系统使用 **参数化查询** 防止 SQL 注入：

```javascript
// 安全：使用参数化查询（? 占位符）
db.prepare('SELECT * FROM users WHERE username = ?').get(username);

// 危险（系统中不存在此写法，仅作反面示例）
// db.exec(`SELECT * FROM users WHERE username = '${username}'`);
```

所有路由中的数据库操作均使用 `better-sqlite3` 的 `prepare().get()` / `prepare().all()` / `prepare().run()` 方法，参数通过占位符传递，天然防止 SQL 注入。

## 9. 性能优化建议

### 索引优化

当前系统已创建的索引覆盖了主要查询场景。如果出现新的慢查询，可通过以下方式分析：

```sql
-- 使用 EXPLAIN QUERY PLAN 分析查询执行计划
EXPLAIN QUERY PLAN
SELECT * FROM emotions WHERE user_id = 1 ORDER BY created_at DESC LIMIT 20;
```

如需添加新索引：

```sql
-- 示例：为情绪类型添加索引（如果频繁按情绪筛选）
CREATE INDEX IF NOT EXISTS idx_emotions_emotion ON emotions(emotion);

-- 示例：共振表的复合索引
CREATE INDEX IF NOT EXISTS idx_resonances_composite
ON resonances(from_user_id, to_user_id, emotion_id);
```

### 定期 VACUUM

SQLite 删除数据后不会自动回收磁盘空间。定期执行 VACUUM 命令回收空间：

```bash
# 手动执行
sqlite3 backend/data/teahaixin.db "VACUUM;"

# 建议每周或每月执行一次
# 注意：VACUUM 会锁定数据库，建议在低峰期执行
```

### WAL 模式

系统已默认启用 WAL（Write-Ahead Logging）模式：

```javascript
db.pragma('journal_mode = WAL');
```

WAL 模式的优势：

| 特性 | 说明 |
|------|------|
| 读写并发 | 允许读操作和写操作同时进行 |
| 更好的写入性能 | 写入操作追加到日志文件，减少磁盘 I/O |
| 崩溃恢复 | 意外断电后自动恢复数据一致性 |

### 查询优化建议

| 优化策略 | 说明 |
|----------|------|
| 限制返回行数 | 所有列表查询均使用 LIMIT 和 OFFSET 分页 |
| 避免 SELECT * | 仅查询需要的字段（系统中部分查询可进一步优化） |
| 合理使用索引 | WHERE 条件中的字段应有索引覆盖 |
| 避免函数索引失效 | 注意 `date()` 等函数可能导致索引失效 |

### 紧急恢复流程

当数据库出现损坏时：

1. **立即停止服务**
   ```bash
   pm2 stop teahaixin-api
   ```

2. **尝试完整性检查**
   ```bash
   sqlite3 backend/data/teahaixin.db "PRAGMA integrity_check;"
   ```

3. **如果完整性检查通过但行为异常**
   ```bash
   # 尝试重建数据库
   sqlite3 backend/data/teahaixin.db ".dump" | sqlite3 backend/data/teahaixin_new.db
   mv backend/data/teahaixin.db backend/data/teahaixin_corrupted.db
   mv backend/data/teahaixin_new.db backend/data/teahaixin.db
   ```

4. **如果完整性检查失败**
   ```bash
   # 从最近的备份恢复
   cp backend/data/backups/teahaixin_最近日期.db backend/data/teahaixin.db
   ```

5. **重新启动服务**
   ```bash
   pm2 start teahaixin-api
   ```
