## 1. 系统架构概述

茶海心遇采用前后端分离架构，整体结构如下：

```
┌─────────────────────────────────────────────────┐
│                    客户端层                       │
├──────────┬──────────────┬───────────────────────┤
│ React    │ PWA 网页版    │ 宣传官网               │
│ Native   │ (webapp/)    │ (website/)             │
│ (frontend/) │            │                       │
└──────┬───┴──────┬───────┴───────────────────────┘
       │          │
       │   HTTP / WebSocket
       │          │
┌──────▼──────────▼───────────────────────────────┐
│              后端服务层 (backend/)                │
├─────────────────────────────────────────────────┤
│  Express.js + Socket.IO                         │
│  ┌──────────────────────────────────────────┐   │
│  │ 路由: generate / user / map / resonance  │   │
│  │       / report                           │   │
│  ├──────────────────────────────────────────┤   │
│  │ 中间件: CORS / JSON / JWT Auth           │   │
│  ├──────────────────────────────────────────┤   │
│  │ 服务: FreeAIService (freeAI.js)          │   │
│  └──────────────────────────────────────────┘   │
├─────────────────────────────────────────────────┤
│  SQLite 数据库 (better-sqlite3)                  │
│  文件: backend/data/teahaixin.db                 │
└─────────────────────────────────────────────────┘
       │
       │ 外部 API 调用
       ▼
┌─────────────────────────────────────────────────┐
│  Pollinations.ai (图片生成)                      │
│  HuggingFace Inference API (可选，情绪分析)      │
│  Pixabay CDN (背景音乐)                          │
└─────────────────────────────────────────────────┘
```

### 技术栈

| 层级 | 技术 |
|------|------|
| 前端（原生） | React Native + Expo + React Navigation |
| 前端（PWA） | 原生 HTML/CSS/JS + Service Worker |
| 前端（官网） | 静态 HTML 单页面 |
| 后端 | Node.js + Express.js |
| 实时通信 | Socket.IO |
| 数据库 | SQLite (better-sqlite3) |
| 认证 | JWT (jsonwebtoken) + bcryptjs |
| AI 服务 | 本地关键词分析 + HuggingFace API (可选) |
| 图片生成 | Pollinations.ai 免费 API |
| 音乐资源 | Pixabay 音频 CDN |

## 3. 后端部署与启动

### 3.1 克隆代码与安装依赖

```bash
# 克隆项目
git clone <仓库地址> teahaixin
cd teahaixin

# 安装后端依赖
cd backend
npm install
```

**主要后端依赖包：**

| 包名 | 用途 |
|------|------|
| express | Web 框架 |
| cors | 跨域支持 |
| socket.io | WebSocket 实时通信 |
| better-sqlite3 | SQLite 数据库驱动 |
| jsonwebtoken | JWT 令牌生成与验证 |
| bcryptjs | 密码加密 |
| axios | HTTP 请求（调用外部 API） |
| dotenv | 环境变量加载 |

### 3.2 环境变量配置

在 `backend/` 目录下创建 `.env` 文件：

```bash
cp .env.example .env  # 如果有示例文件
# 或手动创建
touch .env
```

**.env 文件字段说明：**

| 字段 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `PORT` | 否 | `3001` | 后端服务监听端口 |
| `JWT_SECRET` | 是 | `teahaixin-secret-key-2024` | JWT 签名密钥，**生产环境务必修改为随机强密钥** |
| `HF_TOKEN` | 否 | 空 | HuggingFace API Token，用于可选的 AI 情绪分析增强。不填则使用本地关键词分析 |

**.env 示例：**

```env
PORT=3001
JWT_SECRET=your-super-secret-random-key-here-at-least-32-chars
HF_TOKEN=hf_xxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

> **安全警告：** `JWT_SECRET` 必须在生产环境中替换为强随机字符串。默认密钥仅用于开发环境。

### 3.3 启动服务

```bash
cd backend

# 开发模式启动
npm start

# 或直接运行
node server.js
```

启动成功后将显示：

**验证服务是否正常运行：**

```bash
curl http://localhost:3001/api/health
```

预期返回：

```json
{
  "status": "ok",
  "brand": "茶海心遇",
```

### 3.4 生产环境部署建议

#### 使用 PM2 进程管理

```bash
# 安装 PM2
npm install -g pm2

# 启动服务
pm2 start backend/server.js --name teahaixin-api

# 查看状态
pm2 status

# 查看日志
pm2 logs teahaixin-api

# 设置开机自启
pm2 startup
pm2 save
```

#### Nginx 反向代理配置

```nginx
server {
    listen 80;
    server_name api.teahaixin.com;

    # 强制 HTTPS 跳转
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name api.teahaixin.com;

    ssl_certificate     /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    # API 请求代理
    location /api/ {
        proxy_pass http://127.0.0.1:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket 代理
    location /socket.io/ {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

#### 生产环境注意事项

1. 修改 `JWT_SECRET` 为强随机密钥
2. 配置 HTTPS 证书
3. 设置 CORS 为指定域名（不使用通配符 `*`）
4. 配置防火墙仅开放必要端口
5. 定期备份数据库文件 `backend/data/teahaixin.db`
6. 配置日志轮转

### 4.2 PWA 网页版部署

PWA 网页版位于 `webapp/` 目录，可部署到任何静态文件托管服务。

#### 部署到 GitHub Pages

1. 确保 `webapp/` 目录中包含以下文件：
   - `index.html` - 主页面
   - `app.js` - 应用逻辑
   - `sw.js` - Service Worker
   - `manifest.json` - PWA 配置清单
   - `style.css` - 样式文件（如有）

2. 推送到 GitHub 仓库的 `main` 分支
3. 在仓库 Settings > Pages 中配置源目录
4. 等待自动部署完成

#### 部署到自有服务器（Nginx）

```nginx
server {
    listen 443 ssl;
    server_name app.teahaixin.com;

    ssl_certificate     /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    root /var/www/teahaixin/webapp;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # 缓存静态资源
    location ~* \.(js|css|png|jpg|svg|ico|mp3)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }

    # Service Worker 不缓存
    location /sw.js {
        expires -1;
        add_header Cache-Control "no-store, no-cache, must-revalidate";
    }
}
```

### 4.3 Android APK 构建

使用 Expo EAS Build 构建 Android APK：

```bash
cd frontend

# 登录 Expo 账号
eas login

# 配置构建（首次）
eas build:configure

# 构建 APK（本地开发测试用）
eas build --platform android --profile preview

# 构建 AAB（Google Play 发布用）
eas build --platform android --profile production
```

**构建配置参考（eas.json）：**

```json
{
  "build": {
    "preview": {
      "android": {
        "buildType": "apk"
      }
    },
    "production": {
      "android": {
        "buildType": "app-bundle"
      }
    }
  }
}
```

构建完成后从 Expo 控制台下载 APK 文件。

### 4.4 iOS 构建说明

```bash
cd frontend

# 构建 iOS 应用
eas build --platform ios --profile production
```

**前提条件：**

| 要求 | 说明 |
|------|------|

> **注意：** iOS 应用需通过 App Store 审核或 TestFlight 分发。EAS Build 可在云端完成构建，不需要本地 macOS 设备。

## 5. API 接口文档

### 通用说明

- 基础路径：`http://localhost:3001`（开发环境）
- 请求格式：`Content-Type: application/json`
- 认证方式：Bearer Token（在请求头中携带 `Authorization: Bearer <token>`）
- 通用响应格式：

```json
{
  "success": true,
  "message": "操作说明",
  "data": {}
}
```

- 错误响应格式：

```json
{
  "success": false,
  "message": "错误描述"
}
```

### 5.1 情绪生成接口

|------|------|------|------|
| text | string | 是 | 用户的心情文字，1-1000 字 |

**请求示例：**

```bash
curl -X POST http://localhost:3001/api/generate \
```

#### POST /api/generate/save - 保存生成结果（需登录）

将情绪生成结果保存到用户的历史记录。

**请求头：** `Authorization: Bearer <token>`

**请求参数：**

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| text | string | 是 | - | 心情文字 |
| emotion | string | 是 | - | 情绪类型（happy/sad/angry/peaceful/anxious/romantic） |
| image_url | string | 否 | "" | 生成的图片 URL |
| audio_url | string | 否 | "" | 音频 URL |
| poem_text | string | 否 | "" | 诗意文本 |
| location | string | 否 | "" | 位置描述文字 |
| latitude | number | 否 | 0 | 纬度 |
| longitude | number | 否 | 0 | 经度 |
| is_public | boolean | 否 | true | 是否公开可见 |

**成功响应 (200)：**

```json
{
  "success": true,
  "message": "情绪记录已保存",
  "data": { "id": 42 }
}
```

**错误响应：**

| 状态码 | 场景 | 消息 |
|--------|------|------|
| 400 | 缺少 text 或 emotion | "缺少必要参数" |
| 401 | 未登录或 Token 无效 | "请先登录" |

#### GET /api/user/history - 获取历史记录（需登录）

**请求头：** `Authorization: Bearer <token>`

**请求参数（Query）：**

| 字段 | 类型 | 必填 | 默认值 | 说明 |

## 6. WebSocket 事件说明

茶海心遇使用 Socket.IO 实现实时通信功能，主要用于共振通知的实时推送。

### 连接方式

```javascript
import io from 'socket.io-client';

const socket = io('http://localhost:3001');
```

### 客户端事件

| 事件名 | 方向 | 参数 | 说明 |
|--------|------|------|------|
| `register` | 客户端 -> 服务器 | `userId` (string/number) | 注册用户 ID 与 Socket 的映射关系 |

**示例：**

```javascript
// 用户登录后注册 Socket
socket.emit('register', userId);
```

### 服务器事件

| 事件名 | 方向 | 数据结构 | 说明 |
|--------|------|----------|------|
| `resonance_notification` | 服务器 -> 客户端 | 见下方 | 当有人与你产生共振时推送通知 |

**resonance_notification 数据结构：**

```json
{
  "id": 5,
  "fromUser": {
    "id": 1,
    "username": "茶友小明"
  },
  "emotionId": 15,
  "emotion": "happy",
  "message": "茶友小明 与你产生了情绪共鸣",
  "createdAt": "2026-03-25T10:00:00.000Z"
}
```

### 连接管理

- 服务器在内存中维护 `onlineUsers` Map（userId -> socketId）
- 用户连接时记录日志：`[Socket] 用户连接: <socketId>`
- 用户注册后记录：`[Socket] 用户 <userId> 已注册，在线用户数: <count>`
- 用户断开时自动清理映射关系

## 8. 常见错误排查

### 启动失败

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| `Error: Cannot find module 'better-sqlite3'` | 依赖未安装 | 运行 `npm install` |
| `EADDRINUSE: address already in use` | 端口被占用 | 修改 `.env` 中的 `PORT` 或终止占用端口的进程 |
| `better-sqlite3` 编译失败 | 缺少编译工具 | 安装 `build-essential`（Linux）或 Xcode CLI（macOS） |

### 运行时错误

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| 401 "请先登录" | Token 未携带或已过期 | 检查 Authorization 请求头格式是否正确 |
| 401 "登录已过期" | JWT Token 超过 7 天有效期 | 重新登录获取新 Token |
| 图片生成返回空 URL | Pollinations.ai 服务不可用 | 检查网络连接，等待服务恢复 |
| HuggingFace API 调用失败 | Token 无效或服务限流 | 检查 HF_TOKEN 配置，系统会自动降级为本地分析 |
| 数据库锁定错误 | 并发写入冲突 | 确保 WAL 模式已启用（默认启用） |

### 前端问题

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| Expo 启动失败 | Expo CLI 版本不兼容 | 更新 Expo CLI：`npm install -g expo-cli` |
| PWA 无法安装 | 非 HTTPS 环境 | PWA 安装要求 HTTPS 或 localhost |
| Service Worker 不更新 | 浏览器缓存 | 清除浏览器缓存或在 DevTools > Application 中注销 SW |
| API 请求跨域失败 | CORS 配置问题 | 检查后端 CORS 配置是否包含前端域名 |
